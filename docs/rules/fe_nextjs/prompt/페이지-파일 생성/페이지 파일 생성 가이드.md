# 페이지 생성 및 디렉토리 배치 방식

## 디렉토리 구조

- `src > app > ...`: 페이지 컴포넌트 (라우팅 역할만 수행)
- `src > components > view`: 뷰 컴포넌트 (페이지의 실제 구현)
- `src > components > etc`: 기타 컴포넌트
- `src > components > ui`: UI 컴포넌트 디렉토리

## 컴포넌트 호출 계층

페이지 파일(`app/.../page.tsx`) → 뷰 컴포넌트(`components/view`) → UI 컴포넌트(`components/ui`)

- **페이지 파일**: 디렉토리 구조와 파일명만 정의하고, `components/view`의 뷰 컴포넌트를 호출하는 최소한의 코드만 작성
- **뷰 컴포넌트**: 페이지의 실제 비즈니스 로직과 UI 구현을 담당하며, UI 컴포넌트를 조합하여 구성
- **UI 컴포넌트**: 서비스 도메인과 무관한 재사용 가능한 범용 UI 컴포넌트. 뷰 컴포넌트에서 호출하여 사용
- **기타 컴포넌트**: 페이지에서 기타 용도로 사용하는 컴포넌트
